在 Shell 中用 **环境变量（env）填充模板文件**，常见需求是把模板里的变量如 `${VAR}` 替换为环境变量值。下面给你几种 **高效、常用、生产级方法**，按推荐程度排序。

---

# 1️⃣ 推荐方法：`envsubst`（最简单高效）

`envsubst` 来自 **GNU gettext**，专门用于环境变量替换。

### 模板文件

```text
config.template
```

```text
server:
  host: ${HOST}
  port: ${PORT}

database:
  url: ${DB_URL}
```

### 设置环境变量

```bash
export HOST=0.0.0.0
export PORT=8080
export DB_URL=jdbc:mysql://localhost:3306/test
```

### 替换生成文件

```bash
envsubst < config.template > config.yaml
```

生成：

```yaml
server:
  host: 0.0.0.0
  port: 8080

database:
  url: jdbc:mysql://localhost:3306/test
```

---

### 只替换指定变量（推荐）

避免替换系统变量：

```bash
envsubst '${HOST} ${PORT} ${DB_URL}' < config.template > config.yaml
```

---

# 2️⃣ 使用 `envsubst` + `.env` 文件（CI/CD常用）

`.env`

```text
HOST=127.0.0.1
PORT=8080
DB_URL=jdbc:mysql://localhost:3306/test
```

加载变量：

```bash
export $(grep -v '^#' .env | xargs)
envsubst < config.template > config.yaml
```

---

# 3️⃣ `sed` 替换（灵活但稍复杂）

模板：

```text
server:
  host: ${HOST}
  port: ${PORT}
```

替换：

```bash
sed -e "s|\${HOST}|$HOST|g" \
    -e "s|\${PORT}|$PORT|g" \
    config.template > config.yaml
```

缺点：

* 变量多时维护困难
* 不适合大模板

---

# 4️⃣ `awk` 实现 env 替换（高级）

```bash
awk '{
  while(match($0, /\$\{[A-Za-z_][A-Za-z0-9_]*\}/)) {
    var=substr($0,RSTART+2,RLENGTH-3)
    gsub("\\$\\{"var"\\}", ENVIRON[var])
  }
  print
}' config.template
```

优点：

* 自动替换所有 `${VAR}`
* 不依赖外部工具

缺点：

* 可读性差

---

# 5️⃣ `perl` 方案（性能好）

```bash
perl -pe 's/\$\{(\w+)\}/$ENV{$1}/g' config.template > config.yaml
```

---

# 6️⃣ `gomplate`（企业级模板引擎）

推荐工具：
gomplate

安装后：

```bash
gomplate -f config.tmpl -o config.yaml
```

模板：

```yaml
server:
  host: {{ getenv "HOST" }}
  port: {{ getenv "PORT" }}
```

优点：

* 支持 JSON / YAML
* 支持复杂逻辑

---

# 7️⃣ Kubernetes / Helm 场景

很多 CI/CD 使用：

```bash
envsubst < deployment.yaml.tpl > deployment.yaml
kubectl apply -f deployment.yaml
```

示例模板：

```yaml
image: ${DOCKER_IMAGE}
namespace: ${NAMESPACE}
replicas: ${REPLICAS}
```

# 8️⃣ 企业级最佳实践脚本（推荐）

```bash
#!/usr/bin/env bash
set -e

TEMPLATE=$1
OUTPUT=$2

if [ -f ".env" ]; then
  export $(grep -v '^#' .env | xargs)
fi

envsubst < "$TEMPLATE" > "$OUTPUT"
```

使用：

```bash
./render.sh config.template config.yaml
```
